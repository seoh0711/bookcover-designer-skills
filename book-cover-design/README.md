# book-cover-design

Design compelling book covers — from concept storytelling to typography, visual hierarchy, color systems, and print-ready delivery.

## Skills (8)

- **book-concept-analysis** — Analyze a book's genre, audience, and message to extract visual keywords and emotional cues for cover design.
- **visual-storytelling** — Design covers that tell a story through composition, symbol, metaphor, and guided eye movement.
- **cover-typography** — Select and specify genre-appropriate typefaces with hierarchy, pairing, and legibility at all sizes.
- **color-mood-system** — Build a cover color palette from genre and mood with psychological rationale, HEX, RGB, and CMYK values.
- **layout-composition** — Design the three-panel cover (front, spine, back) with grids, bleed, safe zones, and compositional structure.
- **image-illustration-direction** — Choose the right visual medium and write a brief for photographers, illustrators, or AI image tools.
- **genre-benchmark** — Analyze bestseller cover patterns to find differentiation opportunities and inform strategic design decisions.
- **print-delivery-spec** — Generate print and eBook delivery specifications with publisher submission checklists.

## Commands (4)

- `/book-cover-design:conceptualize` — Analyze a book brief and generate three distinct cover concept directions.
- `/book-cover-design:design-system` — Build the typography, color palette, and layout grid for a chosen concept direction.
- `/book-cover-design:direct-visual` — Choose a visual medium and produce a brief for the creator or AI image tool.
- `/book-cover-design:deliver` — Generate print and eBook delivery checklists and a file package guide.

## Workflow

```
/conceptualize  →  /design-system  →  /direct-visual  →  /deliver
```

Each command builds on the output of the previous one, but each can also be run independently when you have an existing brief or design system.
